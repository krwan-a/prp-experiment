# 听觉语义干扰对双任务中枢加工资源的影响 — 实验操作手册

## 一、指导语

### 1. 正式指导语

```
欢迎参加本实验。

本实验包含两个视觉任务，两个任务同等重要。

任务一：屏幕上方会出现一个 3×3 的字母矩阵，
请判断矩阵中是否包含字母 T。
  · 含有 T → 按【F】键（左手食指）
  · 不含 T → 按【J】键（右手食指）

任务二：屏幕下方会出现一个数字（1–9，不含5），
请判断该数字的奇偶性。
  · 奇数 → 按【F】键（左手食指）
  · 偶数 → 按【J】键（右手食指）

注意事项：
1. 两个任务【都使用 F 和 J 键】，请严格按照顺序：
   【先按 F/J 回答矩阵（任务一），再按 F/J 回答数字（任务二）】
   任务一的反应必须在任务二的反应之前完成，不能颠倒顺序；
2. 全程忽略耳机中的背景声音，无需对声音做任何反应；
3. 看到每个刺激后请立即按键，不要等待下一个刺激出现；
4. 两个任务同等重要，对两个任务都要做到既准确又快速的反应；
5. 每个试次中，若 2300ms 内未做出反应，该试次记为错误。

实验共分两部分，中途有休息。每部分约需 5 分钟。

如有疑问，请在实验开始前告知主试。

准备好后，按【空格键】开始练习阶段。
```

### 2. 练习指导语

```
现在进入练习阶段，共 12 个试次，帮助你熟悉操作流程。

练习阶段不播放背景音频。

流程提示：
  · 屏幕中央出现"+"注视点（500~1000ms 随机）
  · → 上方字母矩阵出现（先按 F/J 判断是否含T）
  · → 下方数字出现（再按 F/J 判断奇偶）
  · → 空屏等待反应（2300ms）
  · → 反馈界面（1000ms）
  · → 空屏（200ms）
  · 必须按顺序：先任务一，再任务二。
  · 每次反应后会看到对错反馈。

如有不明白的地方，现在可以询问主试。

按【空格键】开始练习。
```

## 三、实验流程详细说明

### 1. 实验设计概要

- **实验类型**：2（听觉干扰类型：白噪音、母语无序单字语音）× 3（SOA：50ms、150ms、500ms）被试内重复测量设计
- **因变量**：任务二反应时（RT2，单位ms）、整体正确率
- **被试要求**：在校大学生，视力/矫正视力正常，听力正常，右利手，无语言及认知障碍

### 2. 实验材料

#### 2.1 听觉材料

实验全程被试佩戴耳机，音频双耳播放，统一播放音量为 65 dB SPL，所有音频时长均为 60 秒，格式为 MP3，采样率 44100 Hz。

- **白噪音**：平稳纯白噪音，无语音、语义及节律信息。峰值音量 -15~-18 dB，仅作为纯声学背景刺激。
- **母语无序单字语音**：38个无重复、无语义关联的独立中文单字（石、夏、口、树、灯、北、风、足、雨、中、云、山、书、地、小、日、雪、右、草、东、心、月、笔、左、春、路、高、南、大、耳、天、少、花、门、西、手、冬、水），使用标准普通话合成人声，语速中等偏快，字间停顿稳定，峰值音量 -1 dB。

#### 2.2 视觉材料

所有视觉刺激为 PNG 格式，统一尺寸 960×960 像素，黑色背景、白色字符。

- **任务一（字母矩阵）**：3×3九宫格，目标字母 T，干扰字母 I、J、L。共40张图片（含T矩阵20张、无T矩阵20张）。
- **任务二（数字刺激）**：数字 1–9（不含5），共8张独立图片。

### 3. 实验程序

#### 3.1 准备阶段
1. 主试启动实验程序，输入被试编号、年龄、性别
2. 被试佩戴耳机，调整至舒适音量
3. 屏幕显示正式指导语，被试阅读后按空格键继续
4. 屏幕显示练习指导语，被试阅读后按空格键进入练习

#### 3.2 练习阶段
1. 共 12 个试次，无背景音频播放
2. 每个试次流程参见单试次时序图
3. 每次反应后显示 1000ms 反馈（正确/错误）
4. 练习完成后自动进入正式实验

#### 3.3 正式实验阶段
1. 正式实验共 180 个试次，分为两个 Block，每 Block 90 试次
2. Block 顺序根据被试编号奇偶平衡（奇数：白噪音→语音；偶数：语音→白噪音）
3. 每个 Block 开始前播放对应类型的背景音频（循环播放）
4. Block 间设有休息，被试自主控制休息时长
5. 每个试次的详细时序：
   - 注视点"+"（500~1000ms 随机）→ 短暂空屏（50ms）
   - T1 字母矩阵（350ms，屏幕上方居中）
   - 间隔 SOA（50/150/500ms，随机）后 T2 数字出现（350ms，屏幕下方居中）
   - 空屏等待反应（最长 2300ms）
   - 被试做出双任务按键（先F/J回答T1，再F/J回答T2）
   - 反馈界面（1000ms，仅练习阶段显示）
   - 空屏（200ms）
   - 进入下一试次

#### 3.4 按键映射

| 任务 | 刺激类型 | 判断规则 | 按键 | 使用手指 |
|------|---------|---------|------|---------|
| 任务一 | 字母矩阵 | 含有字母T | F键 | 左手食指 |
| 任务一 | 字母矩阵 | 不含字母T | J键 | 右手食指 |
| 任务二 | 数字 | 奇数 | F键 | 左手食指 |
| 任务二 | 数字 | 偶数 | J键 | 右手食指 |

> **注意**：两个任务均使用 F/J 键，通过按键顺序区分——第一次按键计入任务一，第二次按键计入任务二。

#### 3.5 反应时记录规则
- 任务一反应时：从 T1 刺激 onset 至第一次按键的时间（ms）
- 任务二反应时：从 T2 刺激 onset 至第二次按键的时间（ms）
- 2300ms 内未做出反应，该试次记为错误，反应时数据不记录

### 4. 控制变量

| 变量类型 | 具体变量 | 控制方式 |
|---------|---------|---------|
| 听觉 | 音频音量 | 统一 65 dB SPL |
| 听觉 | 音频时长 | 统一 60 秒 |
| 听觉 | 采样率 | 统一 44100 Hz |
| 视觉 | 图片尺寸 | 统一 960×960 像素 |
| 视觉 | 呈现时长 | 统一 350ms |
| 视觉 | 背景色 | 统一黑色 |
| 视觉 | 字符颜色 | 统一白色 |
| 时序 | SOA | 50/150/500ms 随机呈现 |
| 顺序效应 | Block顺序 | 按被试编号奇偶平衡 |
| 顺序效应 | 试次顺序 | 完全随机排列 |
| 环境 | 实验环境 | 统一实验室环境 |

### 5. 施测规范

1. **被试准备**：被试应在实验前保证充足休息，视力或矫正视力正常，听力正常
2. **实验环境**：实验在隔音或低噪音实验室进行，室内光照恒定
3. **设备要求**：使用台式电脑或笔记本电脑，配备外接键盘和耳机，屏幕分辨率不低于 1920×1080
4. **主试职责**：
   - 实验前确认设备正常运行
   - 向被试说明实验要求
   - 实验过程中观察被试状态
   - 实验结束后确认数据已成功导出
5. **数据管理**：每名被试的实验数据自动保存为独立的 CSV 文件，文件命名规则为 `PRP_{被试编号}_{时间戳}.csv`


## 四、程序操作手册

### 1. 环境要求

- **操作系统**：Windows 10/11，macOS，或 Linux
- **浏览器**：Google Chrome（推荐）、Microsoft Edge、Firefox
- **网络**：部署到 GitHub Pages 后需要网络访问；本地运行无需网络

### 2. 启动程序

#### 2.1 本地测试
直接在浏览器中打开 `index.html` 文件即可运行。

#### 2.2 在线部署（GitHub Pages）
```
1. 将代码推送到 GitHub 仓库
2. 仓库 Settings → Pages → 选择 main 分支部署
3. 等待 1-2 分钟，访问 https://{用户名}.github.io/{仓库名}/
```

#### 2.3 嵌入见数平台
在见数问卷中添加「外部链接」题型，URL 设置为：
```
https://{用户名}.github.io/{仓库名}/?subject_id={$id}
```

### 3. 操作流程

```
程序启动 → 弹出被试信息对话框（编号/年龄/性别）
         → 指导语（空格键继续）
         → 练习阶段（12试次，有反馈）
         → 正式实验第一部分 → 休息 → 第二部分
         → 实验结束，自动下载 CSV 数据文件
```

### 4. 按键说明

| 按键 | 功能 |
|------|------|
| **F** | 任务一：含T / 任务二：奇数 |
| **J** | 任务一：不含T / 任务二：偶数 |
| **空格键** | 指导语翻页、继续 |
| **Esc** | 紧急退出实验 |

### 5. 数据文件说明

实验结束后浏览器自动下载 CSV 文件，包含以下字段：

| 字段名 | 说明 | 数据类型 |
|--------|------|---------|
| subject | 被试编号 | 字符串 |
| age | 年龄 | 数字 |
| gender | 性别 | 男/女 |
| trial | 试次序号 | 数字 |
| blockType | 音频条件类型 | white_noise / speech |
| soaMs | 刺激间隔 SOA | 50 / 150 / 500 |
| hasT | 矩阵是否含 T | 1=含T, 0=不含T |
| digit | 目标数字 | 1/2/3/4/6/7/8/9 |
| t1RtMs | 任务一反应时(ms) | 数字 |
| t1Correct | 任务一正确性 | 1=正确, 0=错误 |
| t2RtMs | 任务二反应时(ms) | 数字（核心因变量） |
| t2Correct | 任务二正确性 | 1=正确, 0=错误 |


## 五、故障说明

### 常见问题及处理方法

| 现象 | 可能原因 | 处理方法 |
|------|---------|---------|
| 页面全黑无反应 | 浏览器兼容问题 | 更换 Chrome 浏览器，刷新页面 |
| 音频不播放 | 浏览器自动播放策略 | 点击页面任意位置后重新加载；检查系统音量 |
| 图片不显示 | 资源路径错误 | 检查 assets 目录是否与 index.html 同级；检查图片文件是否存在 |
| 按键无反应 | 焦点不在实验页面 | 点击页面中央使焦点回到实验窗口；检查是否按了 Esc 退出 |
| 全屏未生效 | 浏览器限制 | 手动按 F11 进入全屏 |
| 反馈结果显示不全 | 文字过长 | 已在代码中设置自动换行，若仍异常可刷新重试 |
| 数据未下载 | 浏览器弹窗拦截 | 检查浏览器是否拦截了下载弹窗，允许下载即可 |
| 页面加载慢 | 图片/音频资源较大 | 首次加载需等待数秒，第二次会因缓存而加快 |
| 触屏按钮无反应（移动端） | 触摸事件冲突 | 刷新页面重试，确保使用浏览器而非应用内嵌视图 |

### 数据异常排查

- **某试次反应时为空**：表示该试次 2300ms 内未收集到有效按键，记为错误
- **某试次 t2RtMs 为空但 t1RtMs 有值**：被试只完成了任务一的反应，未对任务二做出反应
- **正确率普遍偏低**：检查被试是否理解按键规则（F/J 按顺序使用）
- **反应时普遍偏长**：检查被试是否遵循"尽快按键"的指示

### 紧急联系

如遇程序无法解决的故障，请联系实验开发者或主试，
并保留页面控制台的报错信息（F12 → Console 面板截图）。
